### Firestore schema (documented in this file)

All timestamps are Firestore `Timestamp` unless stated. Money fields are
stored as plain numbers in their native currency; conversions happen at
read time via cached FX rates. The `connections` document never stores a
plaintext credential — only an encrypted blob (E2E mode) or an opaque
reference to a KMS-wrapped key (server-key mode).

- [x] `users/{uid}` — per-user settings document
  - `base_currency: string` (ISO 4217, e.g. `"HKD"`)
  - `currency_mode: string` (`"native" | "base"`)
  - `locale: string` (`"en" | "zh_Hant"`)
  - `theme: string` (`"system" | "light" | "dark"`)
  - `app_lock: { enabled: bool, method: "biometric" | "pin", pin_hash?: string }`
  - `created_at: Timestamp`, `updated_at: Timestamp`

- [x] `users/{uid}/connections/{cid}` — broker / exchange link
  - `kind: string` (`"longbridge" | "ibkr" | "futu" | "binance"`)
  - `label: string`
  - `credential_mode: string` (`"e2e" | "server_key"`)
  - `encrypted_blob: string` (base64; present iff `credential_mode == "e2e"`)
  - `kms_ref: string` (opaque ID; present iff `credential_mode == "server_key"`)
  - `status: string` (`"ok" | "error" | "stale"`)
  - `last_sync_at: Timestamp`
  - `created_at: Timestamp`

- [x] `users/{uid}/manual_holdings/{id}` — user-entered position
  - `symbol: string`
  - `name: string`
  - `quantity: number`
  - `cost_basis: number`
  - `currency: string`
  - `notes: string`
  - `created_at: Timestamp`, `updated_at: Timestamp`

- [x] `users/{uid}/alerts/{id}` — alert definition
  - `kind: string` (`"price" | "pnl" | "portfolio_value" | "fx"`)
  - `scope: string` (e.g. symbol `"AAPL"`, or `"portfolio"`, or FX pair `"USD_HKD"`)
  - `threshold: number`
  - `direction: string` (`"above" | "below"`)
  - `active: bool`
  - `cooldown_until: Timestamp | null`
  - `created_at: Timestamp`

- [x] `users/{uid}/alert_events/{id}` — fired-alert log
  - `alert_id: string`
  - `fired_at: Timestamp`
  - `payload: map<string, any>` (snapshot of triggering value)
  - `delivered: bool`

- [x] `users/{uid}/devices/{fcm_token}` — registered push device
  - `platform: string` (`"ios" | "android" | "web"`)
  - `last_seen: Timestamp`
  - `app_version: string`

- [x] `fx_rates/{base}_{quote}` — shared FX cache (document id e.g. `USD_HKD`)
  - `base: string`, `quote: string`
  - `rate: number`
  - `fetched_at: Timestamp`
  - `source: string`

### Security rules

- [x] Allow read/write only on `/users/{uid}/...` when `request.auth.uid == uid` (see `firebase/firestore.rules`)
- [x] `/fx_rates/*` readable by any authed user, writable only by backend service account (custom claim `service_account == true`; Admin SDK bypasses rules in production)
- [x] Deny everything else by default
- [x] Emulator tests covering the rules — `firebase/tests/firestore.rules.test.js`; 10/10 passing under `npm run test:emulator`

### Indexes

- [x] Composite index on `alerts` (active + scope) — both `COLLECTION` and `COLLECTION_GROUP` scopes
- [x] Composite index on `alert_events` (alert_id + fired_at desc) — both scopes; see `firebase/firestore.indexes.json`
